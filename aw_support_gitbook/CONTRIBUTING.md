# Contributing to A+W Support Documentation

Thank you for your interest in improving A+W Support documentation!

## How to Contribute

### Reporting Issues

If you find errors or outdated information:
1. Check if the issue already exists
2. Create a new issue with:
   - Page URL or file path
   - Description of the problem
   - Suggested correction (if applicable)

### Suggesting Improvements

To suggest documentation improvements:
1. Open an issue describing the improvement
2. Explain why it would be helpful
3. Provide examples if possible

### Submitting Changes

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test locally with GitBook
5. Submit a pull request

## Style Guide

### Markdown Formatting

- Use ATX-style headers (`#` not `===`)
- Use fenced code blocks with language identifiers
- Use relative links for internal pages
- Include alt text for images

### Writing Style

- Write in clear, concise language
- Use active voice
- Define technical terms on first use
- Include examples where helpful

### File Organization

- Keep files in appropriate category folders
- Use lowercase with hyphens for filenames
- Update SUMMARY.md when adding pages
- Include frontmatter with description

## Questions?

Contact the A+W Documentation Team for guidance.
