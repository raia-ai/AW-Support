# Mitarbeiterrechte

### Lernziele
*   Manuelle Statusvergabe für ausgewählte Mitarbeiter einrichten.
*   Rechtevergabe per Gruppe steuern.

### Nutzen
*   Sie können die manuelle Änderung des Status auf ausgewählte Mitarbeiter einschränken.
*   Für jeden Mitarbeiter können Sie die Programme von A+W Business auswählen, mit denen er oder sie arbeiten darf.

### Merke

**Zugriffsrecht**
Jedem Mitarbeiter kann das Zugriffsrecht auf bestimmte Programmteile eingeräumt werden. Es kann differenziert werden nach:
*   **Ändern:** Bearbeiten von Datensätzen, z. B. Dokumente
*   **Einfügen:** Anlegen von Datensätzen, z. B. einen Auftrag
*   **Löschen:** Löschen von Datensätzen, z. B. eine Auftragsposition
*   **Ausführen, Lesen:** Nur Lesen von Datensätzen, z. B. Bestellungen

**Gruppe**
Identische Zugriffsrechte können über eine Gruppe erteilt werden.

**Profil**
Über die Anwendung eines Profils auf einen Mitarbeiter oder eine Gruppe kann die Rechtevergabe vereinfacht werden. Das Profil gilt jeweils für alle Programme:
*   **Ohne Rechte:** Für alle Programme werden Sätze angelegt und die Checkboxen Ausführen, Ändern, Einfügen und Löschen werden deaktiviert.
*   **Ausführbar:** Für alle Programme werden Sätze angelegt. Nur die Checkbox Ausführen wird aktiviert. Alle anderen werden deaktiviert.
*   **Mit allen Rechten:** In diesem Fall werden bei der betreffenden Gruppe alle Einschränkungen gelöscht.

**Voreinstellungen**
Keine

## Statusvergabe über Mitarbeiterrechte

Die Mitarbeiter Ihres Unternehmens können unterschiedliche Zugriffsrechte auf die Daten in A+W Business erhalten, so dass sehr fein gesteuert werden kann, welche Bereiche für wen offen oder gesperrt sind.

> **Datenschutz**
> Personenbezogene Daten unterliegen dem Datenschutz. Achten Sie daher darauf, dass diese Daten durch entsprechende Zugriffsrechte vor Missbrauch geschützt sind.

Mit der Rechtevergabe an Ihre Mitarbeiter steuern Sie die Zugriffsmöglichkeiten auf die Programmbereiche von A+W Business bis auf die Ebene der Dialoge und zum Teil auch auf die Felder.

Die vordefinierten Mitarbeitergruppen und Mitarbeiterrechte können bearbeitet und mit neuen ergänzt werden.

Ordnen Sie einen neuen Mitarbeiter einer der Mitarbeitergruppen zu, so erhält er automatisch alle Rechte dieser Gruppe. Soll er modifizierte Rechte erhalten, so können Sie diese speziell für ihn ändern.

Für umfangreiche Modifikationen legen Sie am besten eine neue Gruppe mit den entsprechenden Rechten an.

**Abb. C-127: Rechteverwaltung für Mitarbeitergruppen**

*   **A** Mitarbeitergruppe
*   **B** Auswahl der Rechte
*   **C** Programm
*   **D** Subrechte

Im Beispiel sehen Sie, dass die Gruppe Anwender im Programm NV Auftrag einen Exklusiv-Status vergeben darf.

Standardmäßig sind folgende Mitarbeitergruppen eingerichtet: **Manager, Supervisor, Anwender, Geschäftsleitung**. Jede dieser Gruppen hat unterschiedliche Zugriffsrechte, an die betrieblichen Aufgabenstellung angepasst sind.

Mitarbeiterdaten können nicht gelöscht werden. Scheidet ein Mitarbeiter aus dem Betrieb aus, muss er in der Mitarbeiterverwaltung gesperrt werden.

# Dokumente kopieren

A+W Business verwendet die Kopierfunktion, um aus einem Dokument ein neues Dokument mit einer eigenen Nummer zu erstellen. (Ausnahme: Kopieren aus dem Archiv)

In diesem Themenblock lernen Sie, wie Sie neue Dokumente durch Kopieren erstellen.

Dazu gehören folgende Lerneinheiten:
*   "Neuer Auftrag durch Kopieren" auf Seite C-1506
*   "Teillieferungen" auf Seite C-1517
*   "Anzahlungen" auf Seite C-1535
*   "Reklamationen" auf Seite C-1544
*   "Gutschriften" auf Seite C-1551

## Neuer Auftrag durch Kopieren

### Lernziele
*   Kundenauftrag vollständig kopieren.
*   Kundenauftrag teilweise kopieren.

### Nutzen
*   Wiederkehrende Aufträge können Sie schneller erfassen, indem Sie einen gleichlautenden Auftrag kopieren.
*   Da Sie dann nur wenige Daten anpassen oder neu erfassen müssen, reduzieren Sie fehlerhafte Eingaben.

### Merke

*   **Auftrag:** Aus jedem Auftrag kann durch Kopieren ein neuer Auftrag erzeugt werden. Im neuen Dokument können alle Felder bearbeitet werden.
*   **Dokument modifizieren:** Sie können Dokumente mit ähnlichen oder identischen Positionen kopieren und sowohl die Kopfdaten als auch die Positionen bearbeiten.
*   **Voreinstellungen:**
    *   Utilities
    *   Produktaufbau-/Duplikatsfilter
    *   Änderungsüberwachung

## Kopierfunktion

Wenn Sie die Daten eines Dokumentes ganz oder teilweise in ein anderes Dokument übernehmen möchten, können Sie das Original-Dokument kopieren. Sie haben dabei die Möglichkeit, einzelne Positionen auszuwählen und die Stückzahlen zu bestimmen.

**Abb. C-128: Beispiele für die Verwendung der Kopierfunktion**

| von | nach |
| :-- | :--- |
| **Auftrag** | • Auftrag<br>• Teillieferung<br>• Reklamation<br>• Anzahlung |
| **Angebot** | • Auftrag<br>• Alternativangebot |

Die Kopierfunktion ist für alle Dokumente gleich. Sie wird in diesem Tutorial für Aufträge beschrieben.

Die Kopierfunktion starten Sie direkt aus der Dokumentenverwaltung oder aus dem Nummernverwalter heraus. Das bedeutet, dass Sie das Original-Dokument nicht unbedingt öffnen müssen.

Für Aufträge können Sie die Kopierfunktion zu folgenden Zwecken verwenden:
*   Angebot in Auftrag übernehmen.
*   Auftrag überschreiben, z. B., um Fehler zu korrigieren.
*   Archivierten Auftrag zurück in die Hauptdatenbank schreiben.

In kopierten Dokumenten können Sie sowohl die Daten zu Kunden und Lieferanten als auch die Positionen bearbeiten. Wenn Sie das neue Dokument für einen anderen Kunden erstellen, werden die Kundendaten neu eingelesen und die Preise und Sonderzuschläge neu berechnet.

In der Historie des neuen Dokuments werden Referenzen auf das Quell-Dokument gespeichert, nämlich die Auftrags- und Kundennummer und dessen aktueller Saldo. In der Historie wird die Nummer des neuen Dokuments als Referenz gespeichert.

## Statusvergabe beim Kopieren

Der Status kann beim Kopieren eines Dokuments geändert werden. Diese Funktion kann über ein Mitarbeiterrecht gesteuert werden. Ist das Recht nicht vorhanden, erhalten neue Dokumente immer den Status des Original-Dokuments. Dokumente, die aus dem Archiv wiederhergestellt werden, erhalten den Status des Archivdokuments.

## Anpassungen beim Kopieren

Beim Kopieren eines Dokuments haben Sie die Möglichkeit, eine identische Kopie zu erzeugen oder z. B. einzelne Bestandteile zu ändern:
*   Positionen nicht übernehmen
*   Stückzahlen ändern
*   Kunde ändern
*   Kunde übernehmen aber die Lieferanschrift ändern
*   Lieferant für Bestellpositionen ändern
*   Termine bearbeiten, z. B. Liefertermin, Produktionstermin
*   Texte hinzufügen

## Kopieren von archivierten Dokumenten

Mit der Kopierfunktion können Sie archivierte Dokumente wieder in die Hauptdatenbank holen. Dazu legen Sie fest, dass das Dokument mit der alten Dokumentennummer gespeichert wird.

Wenn Sie ein Dokument in eine neue Dokumentennummer kopieren, wird das Gewicht aktualisiert. Damit ist z. B. gewährleistet, dass neue Dokumente aus archivierten Aufträgen korrekte Gewichte angeben.

Die Produktdaten können Sie wahlweise neu einlesen, wenn die Stammdaten seit der Erfassung des Quelldokuments bearbeitet wurden. Damit ist gewährleistet, dass z. B. die Preise und Warengruppen im neuen Dokument aktualisiert sind.

Wenn Sie einen archivierten Auftrag in eine Gutschrift kopieren, können Sie den Reklamationsgrund und -verursacher angeben. Diese werden in die Gutschrift übernommen und können in der Statistik ausgewertet werden. Gutschriften sind in einer separaten Einheit beschrieben.
⇨ "Gutschriften" auf Seite C-1551

## Dokument vollständig kopieren

In dieser Einheit lernen Sie, wie Sie ein Angebot kopieren, um daraus einen neuen, identischen Auftrag zu erstellen.

### So erstellen Sie aus einem Angebot einen Auftrag
1.  Öffnen Sie das Angebot, das Sie kopieren möchten.
    (Siehe Abb. C-129: Angebot, das in einen Auftrag übernommen werden soll)

2.  Wählen Sie im Menü `Funktionen > Gruppe Dokument > Dokumente kopieren`.

3.  Im Dialog `Dokumente kopieren` wird das Quelldokument (A), z.B. `Angebot`, angezeigt. Wenn Sie die Kopierfunktion nicht vom Angebot aus geöffnet haben, müssen Sie folgende Kriterien einstellen:
    *   Art des Quelldokuments, z. B. Angebot
    *   Nummer des Quelldokuments oder
    *   Nummernverwalter (B) und Markierung des Quelldokuments (A) in der Übersicht.

    **(Siehe Abb. C-130: Angebot nach Auftrag kopieren)**
    *   **A** Quelldokument
    *   **B** Nummernverwalter des Quelldokuments
    *   **C** Zieldokument
    *   **D** Modus
    *   **E** Änderungen
    *   **F** Ziel-Optionen, z. B. Dokumentnummer

4.  Wählen Sie den Dokumententyp (C) für das Zieldokument aus, z. B. von `Angebot` nach `Auftrag`. Wenn Sie ein Dokument aus einem Archiv holen möchten, müssen Sie im Feld `von` das entsprechende Archiv auswählen.

5.  Bestimmen Sie den Modus `Kopieren` (D).

6.  Prüfen Sie die Einstellungen im Bereich `Änderung` (E), z. B. den Status. Aktivieren Sie nur die Checkboxen, bei denen Sie die Einstellung für das Zieldokument ändern wollen.

7.  Wählen Sie die Option (F) für die Dokumentennummer.
    *   Bei **Nummer automatisch ermitteln** wird ein neues Dokument mit einer neuen Nummer erstellt.
    *   Bei **Manuelle Vorgabe der Nummer** können Sie die Positionen aus dem Quelldokument in ein Zieldokument kopieren.
    *   Bei **Nummer aus Quelle übernehmen** wird das (alte) Quelldokument aus dem Archiv zurück in die Hauptdatenbank geschrieben.

8.  Wechseln Sie zum Register `Weitere Optionen Zieldokument`, um die Termine zu prüfen. Das Register ist nur freigeschaltet, wenn Sie die Option `Nummer automatisch ermitteln` gewählt haben.

9.  Wählen Sie im Menü `Start > Ausführen`, um die Aktion zu starten. Die Positionen werden geprüft und kopiert. Dabei werden ggf. Meldungen zu den Zuschlägen und Terminen angezeigt. Bestätigen Sie diese mit [Ja] oder [Nein], um die entsprechenden Vorgänge auszuführen oder zu überspringen. Der neue Auftrag wird gespeichert. Im Register `Kopieren positionsweise` können Sie die Nummer des Auftrags ablesen. Sie können ihn unter `Auftrag` anschließend weiterbearbeiten, z. B. um weitere Positionen hinzuzufügen oder die Termine zu ändern.
    ⇨ "Auftragsposition bearbeiten oder hinzufügen" auf Seite C-1391

10. Schließen Sie den Dialog `Dokumente kopieren`. Der Dialog `Dokumentenverwaltung` mit dem Quell-Dokument wird wieder angezeigt. In der Dokumentenübersicht wird das neue Dokument aufgeführt und mit einem Pfeil im Zeilenkopf gekennzeichnet. In der Historie des Angebots wird der Auftrag als Referenz angezeigt.
    (Siehe Abb. C-131: Angebotshistorie mit Referenz auf neuen Auftrag)

11. Im neuen Auftrag können Sie die Daten anpassen und Positionen ergänzen.
    (Siehe Abb. C-132: Neuer Auftrag, der durch Kopieren erstellt wurde)

## Dokument positionsweise kopieren
Mit dieser Funktion kopieren Sie einzelne Positionen eines Auftrags in ein neues Dokument.

### So kopieren Sie ein Dokument positionsweise
1.  Öffnen Sie den Auftrag, den Sie kopieren möchten.
2.  Wählen Sie im Menü `Funktionen > Gruppe Dokument > Dokumente kopieren`. Der Dialog `Dokumente kopieren` wird geöffnet.
    ⇨ Softwarereferenz, "Dokumente kopieren" auf Seite C-1813
3.  Führen Sie im Register `Kopieren 1:1` alle erforderlichen Schritte aus, wie in der vorausgehenden Handlungsanleitung beschrieben:
    ⇨ "So erstellen Sie aus einem Angebot einen Auftrag" auf Seite C-1509
4.  Wechseln Sie zum Register `Kopieren positionsweise`. Im Bereich `Quelle` werden alle Positionen aus dem Quelldokument aufgeführt.
    (Siehe Abb. C-133: Auftrag positionsweise kopieren; A: Original-Menge, B: Ziel-Menge)
5.  Ändern Sie ggf. in der Spalte `Ziel-Menge (Neu)` (A) die Stückzahl für den neuen Auftrag.
    (Siehe Abb. C-134: Auftrag positionsweise kopieren; A: Ziel-Menge neu, B: Position nicht übernehmen)
6.  Entfernen Sie die Markierung der Position(en), die Sie nicht übernehmen möchten, indem Sie in den Zeilenkopf (B) klicken. Das Häkchen wird danach nicht mehr angezeigt.
7.  Wählen Sie im Menü `Start > Ausführen`, um die Aktion zu starten. Die Positionen werden geprüft und kopiert. Dabei werden ggf. Meldungen zu den Zuschlägen und Terminen angezeigt. Bestätigen Sie diese mit [Ja] oder [Nein], um die entsprechenden Vorgänge auszuführen oder zu überspringen.
8.  Der neue Auftrag wird gespeichert. Sie können ihn anschließend bearbeiten, z. B., um weitere Positionen hinzuzufügen oder die Termine zu ändern.
    (Siehe Abb. C-135: Neuen Auftrag mit geänderten Positionen kopiert; A: Menge im neuen Auftrag, B: Nummer des neuen Auftrags)
    ⇨ "Auftragsposition bearbeiten oder hinzufügen" auf Seite C-1391
9.  Schließen Sie den Dialog. Der Dialog `Dokumentenverwaltung` wird wieder angezeigt. In der Dokumentenübersicht wird das neue Dokument aufgeführt und mit einem Pfeil im Zeilenkopf gekennzeichnet.
    (Siehe Abb. C-136: Neuen Auftrag mit geänderten Positionen als neues Dokument gespeichert)

# Teillieferungen

### Lernziele
*   Teillieferung erfassen.

### Nutzen
*   Sie können die Kapazitäten besser planen, wenn ein Großauftrag über kleinere Teillieferungen abgewickelt wird.
*   Die Teillieferungen aus einem Auftrag können Sie ohne erneute Dateneingabe erfassen.
*   Aufträge mit (mehreren) Teillieferungen können Sie als Teilrechnungen fakturieren.

### Merke

*   **Teillieferung:** In A+W Business wird bei folgendem Sachverhalt eine Teillieferung erfasst:
    *   Auslieferung einer Teilmenge einer Auftragsposition
    *   Auslieferung einer einzelnen Position, wenn im Auftrag mehrere Positionen enthalten sind.
    Durch die Teillieferung wird ein Auftrag vom Typ `Teillieferung` erzeugt.
*   **Dokument:** Für die Teillieferung wird ein eigener Nummernkreis verwendet. Die Teillieferung kann auf unterschiedliche Weise erzeugt werden:
    *   durch Kopieren aus einem Auftrag
    *   durch Erfassen eines neuen Dokuments
*   **Faktura:** Teillieferungen können mit oder ohne Teilrechnung erzeugt werden.
*   **Teilrechnung:** Die Teilrechnung wird über den Druckpunkt erstellt. Sie ist wie die (normale) Rechnung kein eigenes Dokument.
*   **Voreinstellungen:**
    *   **Stammdaten:**
        *   Statusverwaltung
        *   Statuszuordnung
        *   Nummernkreise
        *   Kundendaten
    *   **Firmendaten:**
        *   Register Parameter (Positionen löschen, Datum)
        *   Register Dokumente (Sperrkennzeichen)
        *   Register Lager / EK / EDI (Erfassungsstellenzuordnung)

## Teillieferung zu einem Auftrag

Teillieferungen werden pro Kunde vereinbart. In den Stammdaten des Kunden aktivieren Sie die Funktion und legen zusätzlich fest, ob die Teillieferungen auch fakturiert werden können. Damit ist in der Dokumentenverwaltung automatisch die Funktion `Teillieferung` freigeschaltet.
(Siehe Abb. C-137: Kundenstammdaten – Teillieferung, Teilfakturierung)

Die Teillieferung wird aus einem Auftrag erzeugt, aber der Teillieferschein ist ein eigenes Dokument mit einer eigenen Dokumentennummer, die aus einem eigenen Nummernkreis gezogen wird.

Das Dokument `Teillieferung` wird über die Kopierfunktion erzeugt. Dabei können Sie die Originalmenge des Auftrages sehen, welche Mengen schon geliefert wurden und welche Mengen bestellt sind.

Das Lieferdatum der Teillieferung wird auf die gleiche Weise festgelegt wie das Lieferdatum im Original-Auftrag. Dabei kann neben dem Lieferdatum der Teillieferung auch das Lieferdatum des Original-Auftrags angepasst werden.

Sie können in den Firmendaten festlegen, dass das Datum standardmäßig aus dem Auftrag übernommen wird.
(Siehe Abb. C-138: Firmendaten - Register Parameter)

Außerdem können Sie festlegen, ob die teilgelieferten Mengen aus dem Original-Auftrag gelöscht werden.

## Teillieferungsübersicht

In der Teillieferungsübersicht können Sie sich zu einem Auftrag alle Teillieferungen auflisten lassen. Darin können Sie erkennen, welche Positionen mit welchen Stückzahlen geliefert wurden und welche Teillieferung mit einer Teilrechnung erstellt wurde.

**(Siehe Abb. C-139: Übersicht über die Teillieferungen zu einem Auftrag)**
*   **A** Originalmengen
*   **B** Menge der Teillieferung
*   **C** Teillieferung
*   **D** Gesamtmengen des Originalauftrags

Die Werte zu gelieferten Mengen (B) und ggf. der Betrag der Teilrechnung werden pro Teillieferung angezeigt. Zum Original-Auftrag (D) werden die Summen der teilgelieferten Mengen und der Teilrechnungen angezeigt.

## Faktura

Wenn die Teillieferungen nicht fakturiert werden dürfen, ist der Auftrag so lange für die Fakturierung gesperrt, bis die letzte Teillieferung erstellt ist. Dazu muss in den Stammdaten ein Sperrkennzeichen angelegt werden. Dieses Sperrkennzeichen muss zusätzlich in den Firmendaten eingetragen werden.
(Siehe Abb. C-140: Firmendaten – Register Dokumente)

Wenn Sie mit einem Kunden Teillieferung(en) vereinbart haben, können Sie bei jedem Auftrag entscheiden, ob eine Teillieferung mit einer Teilrechnung fakturiert wird, oder ob die Gesamtrechnung erst mit der letzten Lieferung erstellt wird.

Für die Auslieferung einer Teillieferung wird ein Lieferschein gedruckt. Wenn neben der Teillieferung auch die Teilfakturierung zugelassen ist, können die komplett gelieferten Positionen wahlweise aus dem Original-Auftrag gelöscht werden.
(Siehe Abb. C-141: Firmendaten – Register Parameter)

Im letzten Lieferschein werden dann nur noch die verbliebenen Positionen und Mengen aufgeführt.

### Teilrechnung
Eine Teilrechnung wird aus dem Dokument `Teillieferung` gedruckt (Druckpunkt).
Die Berechnung der Anlieferpauschale ist für Teillieferungen standardmäßig deaktiviert. Im Dialog `Dokument kopieren` können Sie diese Option aktivieren.
(Siehe Abb. C-142: Dokument kopieren – Menü Optionen)

Für die Berechnung der Anlieferpauschale werden die Daten `Kunde`, `Tour` und `Lieferdatum` herangezogen. Wenn z. B. ein Kunde zwei Lieferungen am selben Tag erhält, wird für beide die Anlieferpauschale berechnet. Dies gilt entsprechend auch für Rechnungen, die in einer Sammelrechnung zusammengefasst werden.

Wenn Sie festgelegt haben, dass die Positionen der Teillieferung mit Teilfakturierung aus dem Auftrag gelöscht werden sollen, führt die Abschlussrechnung nur noch die zuletzt gelieferten Positionen auf.

Wenn die Teilfakturierung im Auftrag nicht zugelassen ist, wird der Original-Auftrag erst nach der letzten Teillieferung fakturiert. Dazu kann in den Stammdaten ein Sperrkennzeichen festgelegt werden.

## Teillieferung erfassen
In dieser Einheit lernen Sie, wie Sie eine Teillieferung zu einem Auftrag erfassen.

### So erfassen Sie eine Teillieferung
1.  Öffnen Sie den Auftrag, zu dem Sie eine Teillieferung erfassen möchten.
    ⇨ "Auftrag suchen" auf Seite C-1306
2.  Wechseln Sie ggf. zum Register `Zusatz` und prüfen Sie, ob die Checkbox `Teillieferung` markiert ist. Wenn gleichzeitig eine Teilrechnung erstellt werden soll, muss auch die Checkbox `Teilfaktura` markiert sein.
3.  Wählen Sie im Menü `Funktionen > Gruppe Dokument > Teillieferung`. Der Dialog `Dokumente kopieren` wird im Modus `Teillieferung` geöffnet. Nur die Felder sind freigeschaltet, die für Teillieferungen relevant sind.
    ⇨ Softwarereferenz, "Dokumente kopieren" auf Seite C-1813
4.  Wechseln Sie zum Register `Kopieren positionsweise`.
5.  Überschreiben Sie in der Spalte `Zielmenge (neu)` die Stückzahl der Position, zu der Sie die Teillieferung erfassen wollen.
    **(Siehe Abb. C-143: Teillieferung erstellen)**
    *   **A** Gesamt Menge der Position
    *   **B** Menge aller Teillieferungen
    *   **C** Bereits teilgelieferte Menge
    *   **D** Restmenge im Original-Auftrag
6.  Wenn Sie einen Wert eingegeben haben, wird die Position mit einem Häkchen gekennzeichnet. Wiederholen Sie diesen Schritt für alle Positionen, zu denen Sie eine Teilmenge angeben wollen.
    (Siehe Abb. C-144: Menge der Teillieferung angeben)
7.  Wechseln Sie zum Register `Weitere Optionen Zieldokument`, um die Termine zu bearbeiten. Prüfen Sie den Termin für die Teillieferung und passen Sie ihn ggf. an.
    (Siehe Abb. C-145: Termine für die Teillieferung festlegen)
    Wenn in den Kundenstammdaten mehrere Lieferanschriften hinterlegt sind, werden diese aufgelistet. Eine markierte Lieferanschrift übernehmen Sie mit dem Pfeil nach oben.
8.  Wählen Sie im Menü `Start > Ausführen`, um die Teillieferung zu erstellen. Die Positionen werden geprüft und kopiert. Die Teillieferung wird gespeichert. Die Nummer der Teillieferung wird im Bereich `Ziel` angezeigt. Die Positionen sind mit denselben Positionsnummern aufgeführt wie im Original-Auftrag.
    (Siehe Abb. C-146: Teillieferung erstellt)
9.  Schließen Sie den Dialog. Der Dialog `Dokumentenverwaltung` wird wieder angezeigt. In der Dokumentenübersicht wird das neue Dokument aufgeführt und mit einem Pfeil im Zeilenkopf gekennzeichnet. In der neuen Teillieferung können Sie die Termine und die Lieferanschrift ebenfalls anpassen.
    (Siehe Abb. C-147: Teillieferung)

> **Teillieferung als Dokument öffnen**

# Anzahlungen

### Lernziele
*   Anzahlung als Rechnung erfassen.
*   Anzahlung als Auftragsposition erfassen.

### Nutzen
*   Sie können unterschiedliche Anzahlungen mit und ohne Faktura erfassen.
*   Sie können die Anzahlung für einen Kunden erfassen, mit dem Sie Vorauskasse vereinbart haben.

### Merke

*   **Anzahlung:** In A+W Business wird bei folgendem Sachverhalt eine Anzahlung erfasst:
    *   Anzahlung eines Teils der Auftragssumme
    *   Vorauskasse
    Durch die Anzahlung wird eine Anzahlungsposition oder eine Anzahlungsrechnung erzeugt.
*   **Produkt Anzahlung:** Eine Anzahlung muss als Produkt in den Stammdaten angelegt sein.
*   **Anzahlungsposition:** Die Anzahlungsposition wird in den Auftrag geschrieben und erst mit der Endrechnung fakturiert.
*   **Anzahlungsrechnung:** Für eine Anzahlungsrechnung wird ein Auftrag vom Typ `Anzahlung` erzeugt. Dieser kann fakturiert werden.
*   **Dokument:** Für die Anzahlungsrechnung wird ein eigener Nummernkreis verwendet. Die Anzahlungsrechnung kann auf unterschiedliche Weise erzeugt werden:
    *   durch Kopieren aus einem Auftrag
    *   durch Erfassen eines neuen Dokuments
*   **Voreinstellungen:**
    *   **Stammdaten:**
        *   Nummernkreise
        *   Produkt Anzahlung für Anzahlungsposition
        *   Produktzuordnung Zuschläge (Anzahlung) (optional)

## Fakturierte Anzahlung

Vor der Fertigung eines Auftrags können Sie eine Anzahlung vereinbart haben. Solche Anzahlungen können Sie auf zwei verschiedene Arten erfassen und verbuchen:
*   Anzahlungsrechnung
*   Anzahlungsposition (Diese Form der Anzahlung wird in der nächsten Einheit beschrieben. ⇨ "Anzahlung als Auftragsposition" auf Seite C-1537)

Anzahlungen können als Anzahlungsrechnungen erfasst und automatisch an die Finanzbuchhaltung, das Archiv und die Statistik übergeben werden. Wenn für einen Kunden oder einen einzelnen Auftrag Vorauskasse vereinbart ist, wird eine Anzahlungsrechnung über die gesamte Auftragssumme erstellt.

Das Dokument `Anzahlungsrechnung` wird über die Kopierfunktion erzeugt. Es erhält automatisch eine eigene Dokumentennummer, die aus dem Nummernkreis für Teillieferungen genommen wird. Bis zur Abschlussrechnung des Auftrags können weitere Anzahlungsrechnungen erstellt werden. Jede weitere Anzahlungsrechnung erhält eine eigene Rechnungsnummer.

Im Original-Auftrag wird der Anzahlungsbetrag als negativer Posten aufgeführt, der den Gesamtbetrag und die MwSt. automatisch um den Anzahlungsbetrag reduziert.

Der Anzahlungsbetrag wird auf ein separates Erlöskonto `Anzahlung` gebucht. In der Statistik wird die zugeordnete Warengruppe `Anzahlung` belegt. Erst wenn der Original-Auftrag an die Finanzbuchhaltung und an die Statistik übergeben wird, erfolgt die Entlastung des Erlöskontos und der Warengruppe für die Anzahlung. Das Original-Dokument wird auf den regulären Konten und Warengruppen verbucht. So werden sowohl in der Statistik als auch in der Finanzbuchhaltung alle Konten ordnungsgemäß bebucht.

Die Abschlussrechnung wird aus dem Original-Auftrag gedruckt. Im Rechnungsformular wird nach den einzelnen Positionen die Auftragssumme mit dem Nettobetrag, der MwSt. und dem Bruttobetrag dargestellt. Danach werden alle Anzahlungsrechnungen mit ihren Einzelbeträgen aufgeführt, also mit Nettobeträgen, der jeweiligen MwSt. und dem Bruttobetrag. Als letzte Zeile wird der verbleibende Rechnungsbetrag gedruckt.
⇨ "Druck" auf Seite C-1450

### Anzahlungsrechnung
Vor dem Erstellen des Dokuments `Anzahlung` legen Sie fest, ob in der Anzahlungsrechnung die Positionen aufgeführt werden sollen.
(Siehe Menü: Start > Optionen > Anzahlungsrechnung mit Positionen)
Wenn die Original-Positionen nicht aufgeführt werden sollen, ist in der Anzahlungsrechnung lediglich die Position `Anzahlung` enthalten.

## Anzahlung als Auftragsposition

Sie können eine Anzahlung im aktuellen Auftrag erfassen. Der erfasste Betrag wird jedoch nicht an die FiBu übergeben, sondern im Auftrag wie eine Position aufgeführt. Der Gesamtwert des Auftrags wird dabei nicht um die Anzahlungssumme vermindert.

Der Anzahlungsbetrag wird erst verbucht, wenn aus dem Auftrag eine Rechnung erzeugt wird.

> **Anzahlung - ein Produkt**
> Für die Anzahlungsposition muss in den Stammdaten ein Produkt `Anzahlung` angelegt sein.

## Anzahlungsübersicht

**(Siehe Abb. C-153: Liste der Anzahlungen zu einem Auftrag)**
*   **A** Nummer der Anzahlungsrechnung
*   **B** Anzahlungsposition

Sowohl die fakturierten Anzahlungen (Anzahlungsrechnungen) als auch die Anzahlungspositionen werden in der Anzahlungsübersicht aufgelistet.

# Reklamationen

### Lernziele
*   Reklamation zu einem Auftrag erfassen.
*   Reklamationen nach Verursacher und Grund kennenlernen.

### Nutzen
*   Sie wissen, wie Sie die Reklamation behandeln und für Ersatz sorgen.
*   Sie können die Reklamationen über einen bestimmten Zeitraum auswerten und die Fehlerquellen beseitigen.

### Merke

*   **Reklamation:** In A+W Business werden bei folgenden Sachverhalten Reklamationen erfasst:
    *   Beanstandungen durch den Kunden
    *   Bruch der Ware bei der Lieferung
    *   Bruch der Einheit in der Produktion
    Durch die Reklamation wird ein Reklamationsauftrag erzeugt.
*   **Dokument:** Für die Reklamation wird ein eigener Nummernkreis verwendet. Die Reklamation kann auf unterschiedliche Weise erzeugt werden:
    *   durch Kopieren aus einem Auftrag
    *   durch Erfassen eines neuen Dokuments
*   **Reklamationsauftrag:** Bei kostenlosen Reklamationen werden alle Preiszuschläge und automatischen Zuschläge automatisch gelöscht.
*   **Reklamationskriterien:** Reklamationen können nach folgenden Kriterien ausgewertet werden:
    *   Reklamationsgrund
    *   Reklamationsverursacher (Reklamationsorte)
*   **Voreinstellungen:**
    *   **Stammdaten:**
        *   Nummernkreis
        *   Reklamationsverursacher
        *   Reklamationsgrund
    *   **Firmendaten > Register Archiv:**
        *   Reklamationsverursacher als Pflichteingabe
        *   Default Reklamationsort (-verursacher)

## Reklamationsauftrag

Die Reklamation wird durch Kopieren aus einem Auftrag erzeugt. Dabei können einzelne Positionen des Auftrags oder der gesamte Auftrag übernommen werden.

Durch das Kopieren entsteht ein Reklamationsauftrag, aus dem keine Rechnung erzeugt wird. Reklamationen werden in eigenen Nummernkreisen geführt.

Durch die Reklamation wird ein Reklamationsauftrag erzeugt, der wie jeder andere Auftrag an die Produktion und/oder an den Einkauf weitergegeben werden muss.

Bei kostenlosen Reklamationen werden alle Preiszuschläge und automatischen Zuschläge automatisch gelöscht. Der Ursprungs-VK wird nicht neu berechnet. Er wird in der Reklamationsstatistik zur Auswertung herangezogen.

Wenn in der Reklamation Positionen mit Produktionsdaten oder Bestellartikel enthalten sind, werden diese an die betreffenden Schnittstellen übergeben wie bei einem neuen Auftrag.

## Reklamationsgrund und Reklamationsverursacher
Durch die Angabe eines Reklamationsgrundes und eines Reklamationsverursachers haben Sie die Möglichkeit, Reklamationen statistisch auszuwerten.

Mögliche Reklamationsgründe und Reklamationsverursacher werden in den Stammdaten hinterlegt.
(Siehe Abb. C-158: Reklamationsgrund)

Mögliche Reklamationsverursacher (Reklamationsorte) sind z. B.:
*   **Büro:** die Daten wurden falsch aufgenommen.
*   **Produktion:** die Scheiben wurden falsch produziert.
*   **Fahrer:** die Ware wurde bei der Auslieferung beschädigt.

## Reklamationsübersicht

Zu einem Auftrag werden alle Reklamationen aufgelistet. Die reklamierten Mengen, Brutto-Stückpreise, Rabatte und der Gesamtbetrag werden pro Reklamation angezeigt.
(Siehe Abb. C-159: Reklamationen zu einem Auftrag)

In der Übersicht werden die Details zur markierten Reklamation angezeigt.

> **Reklamation als Dokument öffnen**
> Wenn Sie nach Reklamationen suchen, schränken Sie die Suche im Feld `Typ` auf `Reklamation` ein.
> Sie können sich alle Reklamationen zu einem Auftrag in der Reklamationsübersicht anzeigen lassen.

In dieser Einheit lernen Sie, wie Sie einen Reklamationsauftrag durch Kopieren erzeugen.

### So erfassen Sie eine Reklamation

*   Dokument Gutschrift kennenlernen.

### Nutzen
*   Sie wissen, wie Sie eine Gutschrift erstellen, die sich auf einen bestimmten Auftrag bezieht.

### Merke

*   **Gutschrift:** In A+W Business wird bei folgenden Sachverhalten eine Gutschrift erfasst:
    *   Preisbeanstandungen durch den Kunden
    *   Fehlmengen oder Transportschäden sollen nicht durch Nachlieferung ersetzt werden
    *   Stornierung
*   **Dokument:** Gutschriften sind eigenständige Dokumente. Für die Gutschrift wird ein eigener Nummernkreis verwendet. Die Gutschrift kann auf unterschiedliche Weise erzeugt werden:
    *   durch Kopieren aus einem Auftrag
    *   durch Erfassen eines neuen Dokuments
*   **Dokumententyp:** Zur Dokumentenart `Gutschrift` gehört folgender Dokumententyp:
    *   Wertmäßige Buchung
*   **Voreinstellungen:**
    *   **Stammdaten:**
        *   Nummernkreis
    *   **Firmendaten:**
        *   Register Dokumente

## Dokument Gutschrift

Bei Stornierungen, Transportschäden oder Mängelrügen durch den Kunden können Sie komplette Gutschriften oder Teil-Gutschriften für einen Auftrag erstellen und sofort buchen.

Die Nummern für Gutschriften werden standardmäßig aus dem Nummernkreis für Gutschriften genommen. In den Firmendaten können Sie einstellen, dass sie aus dem Nummernkreis für Rechnungen genommen werden.
(Siehe Abb. C-164: Firmendaten – Register Dokumente)

Das Dokument `Gutschrift` wird normalerweise über die Kopierfunktion erzeugt. Es erhält automatisch eine Referenz auf den Original-Auftrag.

Möglich ist auch eine Gutschrift über eine Pauschalsumme. Dieses Dokument wird neu erfasst. Den Bezug zu einem Auftrag müssen Sie in diesem Fall über die Textfunktion manuell eingeben.

Auch bei Gutschriften können Sie einen Reklamationsgrund und -verursacher angeben. Diese Angaben können über die Statistik ausgewertet werden.

### Wertmäßige Buchung
Dieser Dokumententyp wird bei Gutschriften verwendet. Er wird automatisch beim Erfassen der Gutschrift gesetzt. Mengenwerte wie Stück, Quadratmeter und Laufmeter werden dadurch nicht an die Statistik übergeben.

# Bestellung

In diesem Themenblock lernen Sie, wie Sie Auftragspositionen weiter bearbeiten, in denen Bestellungen enthalten sind.

Dazu gehören folgende Lerneinheiten:
*   "Bestellübergabe" auf Seite C-1558

## Bestellübergabe

### Lernziele
*   Auftragspositionen mit Bestellungen in den Bestellpool stellen.
*   Bestellung im Bestellpool prüfen.
*   Lieferant und Liefertermin ändern.
*   Bestellung an den Einkauf übergeben.

### Nutzen
*   Sie können in einem Auftrag auch Positionen erfassen, zu denen Teile dazugekauft werden müssen. Sie brauchen diese Zukäufe nicht neu zu erfassen, sondern übergeben die entsprechenden Auftragspositionen an den Einkauf.
*   Sie erkennen die Umsetzung des Status einer Auftragsposition durch eine Rückmeldung aus dem Wareneingang. Sie wissen, wie Sie den Auftrag dann weiter bearbeiten müssen.

### Merke

*   **Beschaffungsart:** Die Beschaffungsart eines Produktes ist in den Produktstammdaten hinterlegt. Sie kann für jede einzelne Auftragsposition überschrieben werden. Über dieses Kennzeichen erkennt A+W Business die Bestellposition und leitet automatisch die Bestellung ein.
*   **Bestellübergabe:** Die Übergabe einer Bestellung muss manuell angestoßen werden. Aus dem Bestellpool werden nur die Auftragspositionen mit einem Bestellkennzeichen an den Einkauf übergeben.
*   **Lieferant:** Wenn in einem Auftrag mehrere Positionen mit Bestellkennzeichen erfasst wurden, wird für jeden Lieferanten eine eigene Bestellung erzeugt.
*   **Preisvergleich:** Vor der Übergabe der Bestellung können Sie die Preise der Lieferanten vergleichen, die das Produkt liefern können. Dazu muss die Lieferantenkartei gepflegt werden.
*   **Terminänderung:** Wird im Bestellpool das Anlieferdatum beim Kunden geändert, wird diese Änderung auch in den referenzierten Auftrag übernommen.
*   **Voreinstellungen:**
    *   **Stammdaten:**
        *   Nummernkreise
        *   Lieferantenkartei
    *   **Firmendaten:**
        *   Register Produktion (Profit-Center-Abrechnung)
        *   Register Lager/EK/EDI (Abstandhalter mitbestellen)

## Bestellposition im Auftrag

Bestellungen zu Auftragspositionen können direkt aus den Aufträgen heraus erzeugt werden. Dazu muss mindestens eine Stücklisten-Komponente oder die gesamte Position als Bestellung gekennzeichnet sein.

Bestellungen werden nicht automatisch an den Einkauf übergeben. Sind Bestellartikel in einem Auftrag enthalten, können Sie sich dies durch eine Meldung anzeigen lassen, wenn Sie die Positionserfassung schließen.
(Siehe Abb. C-168: Bestellkennzeichen für Stücklisten-Komponente)

Die Bestellung selbst lösen Sie manuell aus. Die Daten können zunächst in einen Bestellpool übergeben werden, um die möglichen Lieferanten und die aktuellen Einkaufpreise zu prüfen. Für diese Übersicht müssen die Daten in der Lieferantenkartei gepflegt werden.

### Produktkennzeichen
In den Produktstammdaten kann die Beschaffungsart für ein Produkt festgelegt werden:
*   **Bestellartikel (komplett):** Bestellung erfolgt inklusive aller Bearbeitungen, die in der Stückliste erfasst wurden. Besteht z. B. ein VSG aus einem Float 5 mm, einer Folie und einem ESG 5 mm, so wird das VSG als Ganzes bestellt. (Beachten Sie hierzu auch die Abgrenzung zu `Bestellartikel`.)
*   **Bestellartikel:** Nur das betreffende Produkt wird bestellt, ohne den Inhalt der Stückliste (z. B. Bearbeitungen) zu beachten. Besteht wiederum z. B. ein VSG aus einem Float, einer Folie und einem ESG und ist das Kennzeichen `Bestellartikel` nun für das ESG gesetzt, so wird nur das ESG beim Lieferanten bestellt. Das VSG wird in Eigenfertigung produziert.

Dieses Produktkennzeichen wird automatisch in die Auftragsposition übernommen. Im Auftrag kann aber außerdem jeder Position ein Bestellkennzeichen zugeordnet werden.

## Lieferantenkartei

Die Lieferantenkartei dient primär der direkten Bestellübergabe von Aufträgen mit Bestellartikeln an den Einkauf, ohne über den Bestellpool gehen zu müssen. Hierfür ist es wichtig, dass ein Standard-Lieferant festgelegt wird.

**(Siehe Abb. C-169: Lieferantenkartei)**
*   **A** Lieferzeit
*   **B** Rang 1 = Standard-Lieferant
*   **C** Einschränkung bei den Maßen

Wenn Sie für die Produkte verschiedene Lieferanten hinterlegt haben, müssen Sie festlegen, mit welcher Priorität (B) diese in eine Bestellung übernommen werden. Als zusätzliche Auswahlkriterien für die manuelle Zuordnung eines Lieferanten können die Lieferzeit (A) und Einschränkungen bei den Maßen (C) dienen.

Die Daten aus der Lieferantenkartei werden bei der Bestellung folgendermaßen ausgewertet:
*   Werden Bestellungen sofort ausgelöst, dann nimmt das System automatisch den Standard-Lieferanten.
*   Ist kein Standard-Lieferant eingetragen, wird die Bestellposition bzw. der Bestellauftrag an den Bestellpool übergeben. Sie müssen dann den gewünschten Lieferanten auswählen. Als Auswahlkriterium kann dann der Vergleich der Preise und der Lieferdauer dienen.

## Bestellung zu einem Auftrag

Die Bestellung wird anhand des Bestellkennzeichens aus den Produktdaten oder aus der Auftragsposition übergeben. Die Daten können zunächst in einen Bestellpool übergeben werden, um die möglichen Lieferanten und die aktuellen Einkaufspreise zu prüfen.

**(Siehe Abb. C-170: Bestellübergabe)**
*   **A** Bestellpool wird gefüllt
*   **B** Bestellungen werden erzeugt
*   **C** Anfragen werden erzeugt

Wenn eine Bestellung direkt (B) erzeugt wird, ohne dass der Bestellpool gefüllt wurde, wird der Standard-Lieferant als Lieferant für die Bestellung ausgewählt. Im Auftrag kann pro Position und/oder Stücklisten-Komponente ein abweichender Lieferant angegeben werden.

Übergeben werden auch Bearbeitungen, die bei einem komplett bestellten Glas nicht direkt unterhalb des Glases stehen, aber ein Bestellkennzeichen haben.

In der Bestellung kann ein beliebiger Text als Kommission eingefügt werden. Die Vorlage dazu wird als erweiterte Einstellung erfasst und kann Formeln für den Austausch von Kommissionstexten aus dem Auftrag enthalten.

Für jede Bestellung einer Sammelrechnung werden die Rechnungs- und Lieferscheinnummer, Nummern der Auftragsbestätigung, der Position und des Lieferscheins übergeben.

### Statuspunkte
Mit den Statuspunkten 30 und 660 – 680 können Sie für das Erzeugen einer Bestellung einen eigenen Status vergeben. Sie können damit unterscheiden, ob ein Auftrag nur im Bestellpool steht oder ob auch eine Bestellung erzeugt wurde.

## Prüfungen zur Bestellübergabe

Im Dialog `Bestellübergabe` können Sie verschiedene Prüfungen aktivieren, die automatisch durchgeführt werden sollen, bevor die Bestellung geschrieben wird. Dazu gehört z. B. die Kontrolle der Termine.

**(Siehe Abb. C-171: Ablauf der Bestellübergabe)**

### Lieferantenauswahl
Zu jedem Lieferanten können in der Lieferantenkartei auch Maximalwerte für Breite und Höhe erfasst werden. Während der Ermittlung des Lieferanten in der Bestellübergabe werden diese Werte mit den vorliegenden Positionsmaßen verglichen. Nur die Lieferanten werden vorgeschlagen, deren Maße über den Positionsmaßen liegen.
Die Reihenfolge der Lieferanten zeigt an, welcher Lieferant zu den besten Bedingungen liefern kann. Die Rangfolge ist aufsteigend.
⇨ "Bestellung im Bestellpool ändern" auf Seite C-1568

### Interne Aufträge
Wenn durch die Bestellübergabe ein interner Auftrag erzeugt wurde, wird in dem Dokument eine Referenz auf den Kunden-Auftrag angegeben, aus dem der interne Auftrag erzeugt wurde.

## Änderung einer bestellten Position

Wenn nach der Bestellübergabe eine Bestellung existiert, werden alle aufbaurelevanten Datenfelder in der Positionserfassung des Auftrages gesperrt. Die Position kann manuell freigeschaltet und korrigiert werden.
(Siehe Abb. C-172: Gesperrte Position)

Wenn die Werte einer Bestellposition bearbeitet wurden, muss die Bestellung erneut übergeben werden. Damit die Bestellung nicht doppelt geliefert wird, muss der Lieferant über die Änderung informiert werden. Vergessen Sie dabei nicht, dem Lieferanten deutlich zu machen, welche Bestellung neu ist und welche frühere Bestellung durch sie ersetzt wird.

## Bestellungen

### Lernziele
*   Bestellung kennenlernen.
*   Referenzierte Bestellung vs. Lagerbestellung.

### Nutzen
*   Sie kennen den Unterschied zwischen referenzierten und unabhängigen Bestellungen.
*   Sie können sich eine Übersicht zu den Bestellungen anzeigen lassen, die aus einem bestimmten Auftrag erzeugt wurden, und daran erkennen, ob der Auftrag weiter bearbeitet werden kann.

### Merke

*   **Dokument erzeugen:** Bestellungen können auf unterschiedliche Weise erzeugt werden:
    *   Durch die Bestellübergabe werden Bestellungen aus einem Auftrag erzeugt.
    *   Durch das Erfassen einer Bestellung. Diese Bestellungen dienen i. d. R. dazu, die Lagerbestände zu ergänzen. Sie werden in der Schulung zum Thema Einkauf behandelt.
*   **Bestellung:** Bestellungen sind genauso aufgebaut wie Aufträge.
*   **Referenzen:** Durch Referenzen ist gekennzeichnet, aus welchem Auftrag die Bestellung erzeugt wurde.
*   **Bestell-Info:** Zu einem Auftrag werden alle referenzierten Bestellungen angezeigt.
*   **Änderungen:** Wenn in einer referenzierten Bestellung ein Termin geändert wurde, wird die Änderung in den Auftrag zurückgeschrieben.
*   **Voreinstellungen:** Keine

### Dokumentenkopf Bestellung
Bestellungen werden im Rahmen dieser Schulung aus Aufträgen erzeugt, in denen Positionen mit Bestellkennzeichen enthalten sind.

Bestellungen, die unabhängig von Aufträgen erfasst werden, sind ausführlich im Part `Einkauf` beschrieben.

# Dokumentendaten

### Lernziele
*   Funktionen des Dialogs kennenlernen.
*   Liefertermin im Auftrag ändern.
*   Dokumente für gemeinsame Änderungen sammeln.

### Nutzen
*   Termine und Lieferanten können Sie in referenzierten Dokumenten gemeinsam prüfen und ändern.
*   Wenn Sie Ihre Nummernverwalter entsprechend organisiert haben, können Sie Änderungen für alle darin gesammelten Dokumente gemeinsam durchführen.

### Merke

*   **Dialog Dokumentendaten:** Im Dialog `Dokumentendaten` können die Termine für folgende Dokumente und Referenz-Dokumente geprüft werden:
    *   Angebot
    *   Auftrag, Bestellung
    *   Anfrage
    *   Gutschrift
*   **Referenz-Dokumente:** Referenzierte Bestellungen werden nur zu einem einzelnen Auftrag angezeigt. Sie werden nicht angezeigt, wenn die Dokumente eines Nummernverwalters aufgelistet werden.
*   **Gemeinsame Änderungen:** Status, Termine, Tour und Lieferbedingungen können für mehrere Dokumente gemeinsam geändert werden.
*   **Voreinstellungen:**
    *   **Stammdaten:**
        *   Kunden: Faxnummer und/oder E-Mail-Adresse
        *   Texte: Standardtexte für die Kundenbenachrichtigungen
        *   Formularverwaltung: Druckpunkte, Formulare
    *   **Status:** Statuspunkte müssen angelegt und zugeordnet sein.

## Lieferdaten anzeigen

Eine Übersicht über die Daten eines Dokuments steht im Dialog `Dokumentendaten` zur Verfügung. Aus diesem Dialog heraus können Teile der Kopfdaten bearbeitet werden, z. B. der Status oder die Termine.

Sie können dabei wahlweise ein einzelnes Dokument oder mehrere Dokumente gleichzeitig ändern. Die Dokumente können aus einem Nummernverwalter genommen werden oder müssen über die Dokumentennummer gesammelt werden.

Der Dialog bietet außerdem die Möglichkeit, alle referenzierten Bestellungen gemeinsam mit dem Auftrag aufzulisten. In dieser Konstellation betreffen die Änderungen dann auch die referenzierten Bestellungen.

**(Siehe Abb. C-186: Termine für Auftrag mit Bestellpositionen prüfen)**
*   **A** Auftrag mit Bestellpositionen
*   **B** Referenzierte Bestellungen

## Lieferterminkontrolle
Sie können die Liefertermine für Aufträge und für Bestellungen prüfen und ggf. korrigieren. Dies führen Sie in zwei Schritten durch:
*   Im Dialog `Dokumentendaten` prüfen und korrigieren Sie die Termine. Mit diesem Vorgehen werden die Termine automatisch auch im Auftrag und in der Bestellung geändert.
*   Im Dialog `Kundenbenachrichtigung` (Lieferterminkontrolle) benachrichtigen Sie den Kunden über die Verzögerung. Für die Benachrichtigung des Kunden können Sie auch die Möglichkeiten im Menü `Kommunikation` nutzen - oder einfach anrufen.

Über die Lieferterminkontrolle können Bestellungen geprüft werden. Bei ausstehenden Lieferungen können die Termine geändert und der Kunde direkt benachrichtigt werden.

## Dokumentenansicht
Die Dokumentenansicht kann aus dem Dialog `Dokumentendaten` heraus angezeigt und gedruckt werden. Damit können Sie die Details eines Dokuments prüfen, ohne den Dialog `Dokumentendaten` zu verlassen.

Wenn Sie die Dokumentenansicht drucken, werden nicht die hinterlegten Formulare verwendet und der Status des Dokuments wird nicht erhöht. Dieser Druck kann nicht mit dem "offiziellen" Druck verwechselt werden, da die wesentlichen Kopf- und Fußdaten nicht wiedergegeben sind.
⇨ "Formular- und Etikettendruck" auf Seite C-1451

## Liefertermin prüfen und ändern

Zu dieser Lerneinheit gibt es folgende Handlungsanleitungen:
*   "So prüfen und ändern Sie referenzierte Dokumente" auf Seite C-1583

### So prüfen und ändern Sie referenzierte Dokumente
1.  Wählen Sie `Dokumente > Dokumentendaten`. Der Dialog `Dokumente` wird geöffnet.
2.  Stellen Sie sicher, dass die Checkbox `Nummernverwalter` nicht markiert ist. Das Feld für die Auftragsnummer ist nur freigeschaltet, wenn keine Nummernverwalter angezeigt werden.
3.  Geben Sie die Auftragsnummer ein und übernehmen Sie die Daten mit `<Enter>`.
    (Siehe Abb. C-187: Termine für Auftrag mit Bestellpositionen prüfen)
4.  Die Kopfdaten des Auftrags und der Bestellung(en) werden angezeigt. Passen Sie den Liefertermin (B) an, den der Lieferant für die Bestellung angegeben hat.
5.  Passen Sie das Datum für den Versandtag (A) und/oder für die Anlieferung beim Kunden an. Wenn Sie die Termine über den Kalender auswählen, können Sie die Tourentage für den Kunden berücksichtigen.
    (Siehe Abb. C-188: Geänderte Termine)
6.  Wählen Sie im Menü `Start > Speichern`, um die Daten zu speichern. Die neuen Termine werden gespeichert und in die zugehörigen Dokumente übernommen. Sie können jetzt den Kunden über den geänderten Termin informieren.

### So benachrichtigen Sie den Kunden über neue Liefertermine
> **Voraussetzung**
> In den Stammdaten des Kunden muss eine E-Mail-Adresse hinterlegt sein. Im Auftrag muss im Bereich `Anschrift` die Checkbox `Mail` aktiviert sein.

1.  Wählen Sie `Dokumente > Kundenbenachrichtigung`. Sie können diesen Dialog auch aus einem Auftrag heraus öffnen. Bei diesem Weg können Sie das Lieferdatum des aktuellen Auftrags jedoch nicht mehr ändern, da der geöffnet ist und damit gesperrt.
    **(Siehe Abb. C-189: Kundenbenachrichtigung)**
    *   **A** Modus Aufträge oder Bestellungen
    *   **B** Einschränkung auf Nummernverwalter
    *   **C** Einschränkung auf Status
    *   **D** Einschränkung auf Datum
2.  Wählen Sie den Modus (A) `Aufträge` oder `Bestellungen`. In diesem Beispiel soll ein Kunde über die Lieferterminverschiebung seines Auftrags informiert werden.
    ⇨ Softwarereferenz, “Kundenbenachrichtigung (Lieferterminkontrolle)" auf Seite C-1848
3.  Schränken Sie die Auswahl auf einen Nummernverwalter (B) oder einen Status (C) ein. Wenn in dem Nummernverwalter sehr viele Dokumente aufgelistet sind, können Sie die Anzeige auf ein Lieferdatum (D) einschränken.
4.  Wählen Sie im Menü `Start > Suchen`, um in die Daten einzulesen.
5.  Markieren Sie im Register `Terminverschiebung` den Auftrag (C), zu dem Sie den Liefertermin beim Kunden ändern müssen, und prüfen Sie den neuen Termin (A). Sie können mehrere Aufträge markieren und einen neuen Liefertermin festlegen. In diesem Fall können Sie die Tour nur ändern, wenn alle Aufträge mit derselben Tour ausgeliefert werden.
    (Siehe Abb. C-190: Liefertermin beim Kunden ändern)
6.  Wählen Sie im Menü `Start > Speichern`, um die Daten zu speichern.
7.  Wählen Sie im Menü `Drucken > Drucker`. Die Benachrichtigung wird angezeigt und kann entsprechend dem Ausgabeformat gespeichert und versendet werden.

### So sammeln Sie Dokumente für gemeinsame Änderungen

2.  Wählen Sie im Feld `Dokument` die Dokumentenart aus, z. B. `Auftrag`.
3.  Stellen Sie sicher, dass die Checkbox `Nummernverwalter` nicht markiert ist.

7.  Wählen Sie im Menü `Start > Speichern`, um die Daten zu speichern. Die neue Tour wird in alle aufgelisteten Aufträge übernommen.

# Angebote

In diesem Themenblock lernen Sie, wie Sie Angebote und Alternativen zu einem Angebot erfassen.

Dazu gehören folgende Lerneinheiten:
*   "Angebotserfassung" auf Seite C-1590
*   "Angebotsverfolgung" auf Seite C-1604
*   "Angebotsstatistik" auf Seite C-1610

## Angebotserfassung

### Lernziele
*   Unterschied zwischen Angebot und Auftrag kennenlernen.
*   Unterschiede zwischen alternativem Angebot und alternativer Position kennenlernen.
*   Alternative Position erfassen.
*   Zu einem Angebot ein vollständiges Alternativangebot erstellen.

### Nutzen
*   Mit einem alternativen Angebot können Sie Vorschläge zu den Wünschen eines Kunden erfassen und später diejenigen Positionen in einen Auftrag übernehmen, für die sich der Kunde entschieden hat.
*   Sie müssen die Daten aus Angeboten nicht erneut erfassen, wenn Sie einen entsprechenden Auftrag erfassen wollen.

### Merke

*   **Angebot:** In A+W Business wird bei folgenden Sachverhalten ein Angebot erfasst:
    *   Kostenvoranschlag für den Kunden
    *   Alternative Vorschläge zu einem zukünftigen Auftrag
    *   Preisliche Alternativen beim Austausch von Gläsern, Sprossen, Modellen und Bearbeitungen
*   **Dokument:** Angebote sind eigenständige Dokumente. Für die Angebote wird ein eigener Nummernkreis verwendet. Das Angebot kann auf unterschiedliche Weise erzeugt werden:
    *   durch Kopieren aus einem Auftrag
    *   durch Erfassen eines neuen Dokuments
*   **Alternativangebot:** Alternativangebote werden im Nummernkreis der Angebote verwaltet.
*   **Voreinstellungen:**
    *   **Stammdaten:** Nummernkreise
    *   **Firmendaten:**
        *   Register System (Wiedervorlage)
        *   Register Produktion (Optimierung)

## Angebot und Alternativen
Zu Beginn eines kaufmännischen Prozesses steht ein Angebot. Für dieses erfassen Sie alle Daten mit den gewünschten Spezifikationen. A+W Business prüft dabei bereits während der Eingabe, ob die erfassten Produkte im Lager vorrätig sind oder angefertigt werden müssen und ob die hinterlegten Restriktionen verletzt werden. So wird sichergestellt, dass die angebotenen Positionen tatsächlich produziert werden können.

Angebote werden auf die gleiche Weise erfasst wie Aufträge.
⇨ Tutorial 2, "Auftrag erfassen" auf Seite B-486

### Alternative Positionen
Zu den Positionen können alternative Positionen erfasst werden, die nicht mit in die Angebotssumme gerechnet werden.

Hat der Kunde sich für ein Angebot entschieden, können Sie mit der Kopierfunktion das Angebot in einen Auftrag umwandeln und dabei die gewünschten Positionen auswählen.

### Alternativangebot
Neben den alternativen Angebotspositionen können Sie auch alternative Angebote erzeugen. So können Sie beispielsweise Gläser austauschen, Modelle, Sprossen und Bearbeitungen zu Positionen ändern, hinzufügen oder löschen.

Wenn Sie ein Alternativangebot erstellen, werden die Daten aus dem Dokumentenkopf automatisch übernommen. Sie müssen also lediglich die Positionsdaten bearbeiten. Das Alternativangebot kann als separates Dokument gespeichert werden.

In einem Überblick können Sie sich anzeigen lassen, welche Alternativangebote zu einem Angebot erstellt wurden.

### Wiedervorlage
Im Dialog `Wiedervorlage` können Sie sich alle Angebote anzeigen lassen, die innerhalb eines bestimmten Zeitraums wieder vorgelegt werden sollen.
(Siehe Abb. C-191: Automatische Meldung beim Systemstart)

Sie können jedem Angebot einen Termin zur Wiedervorlage mitgeben, an dem Sie automatisch erinnert werden, sich erneut mit dem Kunden in Verbindung zu setzen.

Die Wiedervorlage wird ausführlich in einer separaten Einheit erklärt.
⇨ "Angebotsverfolgung" auf Seite C-1604

### Angebotsoptimierung
Bei ungewöhnlichen Scheibengrößen oder Mengen können Sie ein Angebot probehalber an die Produktion übergeben, um den Verschnitt berechnen zu lassen. Aus den zurückgemeldeten Daten können Sie dann die Preise im Angebot neu kalkulieren.
Die Angebotsoptimierung wird in der Schulung zum Part `Fertigung` behandelt.

### Alternativangebotsübersicht
Zu einem Angebot werden alle Alternativangebote aufgelistet. Details zu den Alternativen werden angezeigt, wenn die jeweilige Alternative markiert ist.
(Siehe Abb. C-192: Übersicht über die Alternativangebote zu einem Angebot)

Alternative Positionen werden in dieser Übersicht nicht aufgeführt. Wenn Sie in einem Angebot alternative Positionen erfasst haben, können Sie diese im Angebot selbst an der blauen Schrift erkennen.

## Alternative Position im Angebot erfassen
In dieser Einheit lernen Sie, wie Sie ein Angebot mit alternativen Positionen erfassen. Den Dokumentenkopf und die Positionen erfassen Sie in gleicher Weise wie im Auftrag. Auf diese Bedienschritte wird hier daher nicht nochmal eingegangen.

### So erfassen Sie eine alternative Position
1.  Wählen Sie `Dokumente > Angebot > Angebot auswählen > Register Positionen` oder `<F9>`. Die Positionserfassung wird geöffnet.
    ⇨ Softwarereferenz, "Dokument - Positionen" auf Seite C-1724
2.  Erfassen Sie die Alternativposition genauso, wie Sie jede andere Position erfassen.
    (Siehe Abb. C-193: Alternative Position erfassen)
3.  Markieren Sie im Bereich `Positionen` die Alternativposition und wechseln Sie zum Register `Zusatz`.
4.  Wählen Sie die Position (A) aus, zu der Sie die Alternativposition erfasst haben.
    **(Siehe Abb. C-194: Alternative Position speichern)**
    *   **A** Auswahl der Bezugsposition
    *   **B** Alternative Position
5.  Wählen Sie im Menü `Start > Speichern`, um die Daten zu speichern. Die alternative Position wird gespeichert. Die alternativen Positionen werden mit orangefarbenem Hintergrund dargestellt und im Ausdruck gekennzeichnet.
6.  Schließen Sie die Positionserfassung, um zum Register `Summen` zu wechseln. Im Gesamtpreis wird die alternative Position nicht berücksichtigt.
    (Siehe Abb. C-195: Preis der Alternativposition wird nicht ins Angebot übernommen)
    Im Register `Summen` beziehen die Felder in den Bereichen `Kosten/Deckungsbeitrag`, `Kreditlimit` und `Wertvolumen` alternative Positionen nicht mit ein.
7.  Prüfen Sie in den Kopfdaten im Register `Dokument` das Datum im Feld `Vorlage`, damit Sie automatisch daran erinnert werden, das Angebot weiterzuverfolgen.
8.  Wählen Sie im Menü `Start > Speichern`, um die Daten zu speichern.

## Alternatives Angebot erfassen
In dieser Einheit lernen Sie, wie Sie ein Alternativangebot zu einem bestehenden Angebot erfassen.
Zu dieser Lerneinheit gibt es folgende Handlungsanleitungen.
*   "So erfassen Sie alternative Gläser oder Abstandhalter" auf Seite C-1596
*   "So erfassen Sie eine alternative Sprossenkonstruktion" auf Seite C-1600

### So erfassen Sie alternative Gläser oder Abstandhalter
1.  Öffnen Sie das Angebot, zu dem Sie ein Alternativangebot erfassen wollen.
    (Siehe Abb. C-196: Original-Angebot)
2.  Wählen Sie im Menü `Funktionen > Gruppe Dokument > Alternativen`.
3.  Markieren Sie die Position(en), die Sie in das Alternativangebot übernehmen möchten.
    **(Siehe Abb. C-197: Alternatives Angebot erstellen)**
    *   **A** Originalnummer nicht übernehmen
    *   **B** Markierte Position
    *   **C** Kopieren
    > **Dokumentennummer für Alternativen**
    > Wenn Sie die Checkbox `Originalnummer gleich Alternative` (A) markiert haben, wird das Original-Dokument überschrieben. Wenn Sie ein neues Dokument als Alternative zum Original erstellen möchten, darf die Checkbox nicht markiert sein. Das neue Dokument erhält dann eine neue Nummer.
4.  Klicken Sie auf [Kopieren] (C) und prüfen Sie im Register `Alternative`, ob alle Positionen aufgeführt sind, die Sie übernehmen möchten.
5.  Markieren Sie im Register `Original` die Position, zu der Sie eine Alternative erfassen möchten. Um alle Markierungen zu entfernen, können Sie auf [Keine] klicken und dann die gewünschte(n) Position(en) einzeln markieren.
6.  Wählen Sie den Modus (A), z. B. `1. Glas`, wenn Sie für diese Stücklisten-Komponente eine Alternative angeben möchten. Sie können zu jeder Position mehrere Modi wählen und Alternativen eintragen. Die folgenden Handlungsschritte müssen für jeden Modus einzeln ausgeführt werden.
    **(Siehe Abb. C-198: 1. Glas tauschen)**
    *   **A** Modus auswählen
    *   **B** Glas, das ersetzt werden soll
    *   **C** Glas, das eingebaut werden soll
    *   **D** Änderung starten
    > **Modus Modell, Sprosse und Bearbeitung**
    > Wenn Sie den Modus `Modelle/Bearbeitungen` oder `Sprossenkonstruktion` wählen, wird die Schaltfläche [Definition] freigeschaltet.
    > ⇨ "So erfassen Sie eine alternative Sprossenkonstruktion" auf Seite C-1600
7.  Tragen Sie im Feld `Suchen von` (B) die Produktnummer ein, zu der Sie eine Alternative angeben möchten, z. B. `1004` für das 1. Glas einer ISO-Einheit.
8.  Tragen Sie im Feld `Einfügen/Ersetzen durch` (C) die alternative Produktnummer ein, z. B. `1005`.
9.  Bestätigen Sie die beiden Werte mit [Einfügen/Ersetzen] (D). Sie können die Preise für die Alternative sofort mit [Preisberechnung] berechnen lassen. Sie können die Preise auch neu berechnen lassen, nachdem Sie alle Alternativen angegeben haben.
    > **Generelle Ersetzungen**
    > Wenn Sie alle Positionen markiert haben und den Modus `Hauptprodukt` gewählt haben, wird ein Glas in allen Positionen ersetzt, in denen die entsprechende Produktnummer vorkommt. Bei dem Beispiel in Schritt 7/8 würde in allen Positionen das Produkt `1004` durch das Produkt `1005` ersetzt, also auch in VSG-Scheiben oder die einfachen Gläser.
    > Wenn Sie das Feld `Suchen von` leer lassen, werden alle Gläser in allen markierten Positionen durch das alternative Produkt ersetzt.
10. Wiederholen Sie die Schritte 3 bis 9 für den nächsten Modus der markierten Position oder für andere Positionen. Sie können als Alternative auch andere Preise und/oder Rabatte auswählen und diese mit [Einfügen/Ersetzen] speichern.
    > **Alternativen für markierte Positionen**
    > Achten Sie darauf, dass nur die Positionen markiert sind, die im Alternativangebot mit geänderten Stücklisten-Komponenten aufgeführt werden sollen. Die Markierung einer Position wird nicht automatisch entfernt, wenn Sie eine andere Position markieren.
11. Wechseln Sie zum Register `Alternative`, um den aktuellen Stand der Alternativen zu prüfen.
    (Siehe Abb. C-199: 1. Glas getauscht)
12. Wählen Sie im Menü `Start > Ausführen`, um das Alternativangebot zu erstellen. Beantworten Sie die Abfragen und Meldungen. Das alternative (neue) Angebot wird erstellt. In der Titelzeile wird die neue Nummer angezeigt.
13. Schließen Sie den Dialog. Die Dokumentenverwaltung wird wieder im Vordergrund angezeigt. In der Dokumentenübersicht wird das neue Dokument aufgeführt und mit einem Pfeil im Zeilenkopf gekennzeichnet. Wenn Sie das Original-Dokument überschrieben haben, werden die neuen Daten angezeigt.

### So erfassen Sie eine alternative Sprossenkonstruktion
1.  Führen Sie die Schritte 1 bis 5 wie in der vorhergehenden Handlungssequenz aus.
2.  Wählen Sie den Modus (B), z. B. `Sprossenkonstruktion`. Die Schaltfläche [Definition] wird freigeschaltet.
    **(Siehe Abb. C-200: Sprossenkonstruktion ändern)**
    *   **A** Markierte Position
    *   **B** Modus
3.  Geben Sie die Produktnummer der Sprossen an, die ersetzt werden sollen.
4.  Klicken Sie auf [Definition], um zum Register `Sprossen` zu wechseln. Wenn Sie ein Modell oder die Bearbeitung ändern wollen, wird je nach Modus zum entsprechenden Register gewechselt.
